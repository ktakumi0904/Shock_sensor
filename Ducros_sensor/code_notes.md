## 設計上の工夫

- **グローバル `SCRIPT_DIR`**: 実行場所ではなくスクリプト自身の位置を基準。どこから呼んでも同じ動作。
- **設定ファイル方式**: パラメータ変更のたびにコードを触らず済む。コードは固定。
- **自動除外機能**: 出力 `*_ducros.vtu` を自動検出のリストから除外。繰り返し実行でも二重処理なし。
- **関数分割**: `process_one()` の分離で複数ファイル処理が素直。
- **ヘルパ関数の充実**: `load_config`, `resolve_input_files`, `make_output_path` により各責務を明確化。
