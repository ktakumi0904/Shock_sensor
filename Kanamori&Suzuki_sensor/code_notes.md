## 3. KS センサー本体 `kanamori_suzuki_sensor` (186–261 行)

このスクリプトの中心。

- 186: 関数定義。`gamma` (比熱比) を引数に取るのが特徴。
- 187–206: docstring。原理と返り値の説明。
- 207: 近傍リストを 1 回だけ構築 (以降の勾配計算で使い回す)。

### マッハ数・音速 (209–213 行)

- 210: 速度の大きさ `speed = sqrt(u²+v²)`。
- 211: 音速 `a = sqrt(gamma·p/rho)`。負値や 0 割を `maximum` で保護。
- 212: マッハ数 `mach = speed / a`。
- 213: 超音速フラグ `supersonic = (mach > 1)`。**評価対象を決める鍵**。

### 圧力勾配 (215–217 行)

- 216: 圧力の勾配 `∇p` を計算 (衝撃波面の法線方向の推定に使う)。
- 217: 勾配の大きさ `gradp = |∇p|`。

### 法線マッハ数 Mn (219–232 行)

- 221: `cos(α) = (V·∇p)/(|V||∇p|)`。流れが圧力増加方向へ向かう度合い。
- 223–229: **弱圧力勾配点の保護**。一様流では `∇p≈0` で `cos(α)` がノイズ化するため、
  超音速領域の最大勾配の 1% 未満の点は `cos(α)=1` (超音速のまま) として扱う。
  これは衝撃波の経験的閾値ではなく、方向ベクトルが不定な点を除く数値的保護。
  - 226–227: 超音速領域の代表的な勾配の大きさ `gradp_ref` を取得。
  - 228: 弱勾配フラグ `weak`。
  - 229: 弱勾配点の `cos(α)` を 1 に置換。
- 231: 法線方向の流入マッハ数 `Mn = M·cos(α)`。
- 232: `f = Mn - 1`。**これが衝撃波内部で 0 を横切る**。

### ゼロクロス検出 (234–252 行)

衝撃波の検出ループ。

- 239–240: 節点数と結果配列の初期化。
- 241: 全節点をループ。
- 242–243: **評価は超音速 (M>1) かつ圧縮側 (cos(α)>0) の点のみ**。
  膨張波 (cos(α)<0) はここで除外される。
- 244–246: 近傍を取得 (近傍が無ければスキップ)。
- 247: 自分の `f` 値 (超音速なので正)。
- 248: 近傍の `f` 値。
- 249: **`f[j] < 0` の近傍があるか** = 近傍に法線方向が亜音速 (Mn<1) の点がある。
  これが「超音速→亜音速の境界 = 衝撃波を挟んでいる」ことを意味する。
- 250–252: 符号反転があれば、`f` のジャンプの大きさに圧力勾配の強さを掛けて指標とする。

### 正規化と出力 (254–261 行)

- 254–257: 超音速領域内の最大値で `ks_phi` を 0～1 に正規化。
- 259: `valid` = 超音速フラグ (評価対象だったか)。
- 260: `mask` = `ks_phi>0` の 0/1 マスク。
- 261: `ks_phi`・`mask`・`mach`・`valid` を返す。

## 実装上の判断メモ

- **法線マッハ数方式を採用した理由**: 当初「マッハ線方向の圧力勾配の符号反転」で
  実装したが、垂直衝撃波 (圧力勾配が流れ方向に揃う) で両マッハ線方向への射影が
  同符号になりゼロクロスが起きず、検出できなかった。衝撃波面の法線方向の流入
  マッハ数 `Mn` が音速 (Mn=1) を横切るという、より一般的で頑健な定式化に変更した。
- **弱勾配保護の必要性**: 一様超音速流では `∇p≈0` のため法線方向が不定となり、
  `cos(α)` がノイズ化して偽検出を生む。物理的に「圧力勾配が無い領域は衝撃波面の
  向きが定義できない=衝撃波でない」ため、弱勾配点を保護するのは妥当。
- **本実装の位置づけ**: KS 法の核心 (特性線収束 = 法線マッハ数の音速横断) を
  再現した実用版。原論文の接触不連続/膨張波の厳密な区別までは含まない。
