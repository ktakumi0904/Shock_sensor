# Kanamori-Suzuki 衝撃波センサー (2D 翼まわり / SU2 後処理)

SU2 で計算した 2D 翼まわりの流れ場 (`.vtu`) に対し、特性曲線理論に基づく
**Kanamori-Suzuki (KS) センサー** (Kanamori & Suzuki, JCP 2011) を後処理として
適用し、衝撃波領域を検知するスクリプトです。Ducros センサーとの比較ベースライン、
衝撃波セグメンテーションの参照ラベル生成を想定しています。

## Ducros センサーとの違い

| 観点         | Ducros                          | Kanamori-Suzuki                       |
| ------------ | ------------------------------- | ------------------------------------- |
| 原理         | 圧縮 (div u<0) と渦度の比       | 特性線理論 (法線マッハ数の音速横断)   |
| 経験的閾値   | 必要 (threshold)                | 不要 (Mn=1 が物理的境界)              |
| 必要な物理量 | 速度のみ                        | 速度 + 圧力 + 密度 (マッハ数を計算)   |
| 適用範囲     | 全域                            | **超音速領域 (M>1) のみ**             |
| 膨張波       | 圧縮のみ拾えば除外可            | 流れの向きで自然に除外                |

### ★ 最重要の制約 ★

マッハ角 `mu = arcsin(1/M)` は **M > 1 でしか実数になりません**。
そのため KS センサーは**超音速領域でのみ有効**です。翼まわりの遷音速流では、
超音速ポケット内部だけが検出対象になり、亜音速領域は評価対象外
(`ks_phi = 0`, `supersonic = 0`) となります。Ducros と結果を比較する際は、
この適用範囲の違いを必ず考慮してください。

## 使い方

SU2 側で ParaView 形式を出力します。KS センサーは速度に加えて**圧力と密度**が
必要なので、`VOLUME_OUTPUT` に `PRESSURE`, `DENSITY` が含まれていることを確認してください。

```bash
python kanamori_suzuki_sensor.py
```

引数は不要です。`config.yaml` を読み、同じディレクトリの `.vtu` を自動検出して処理します。

## 検出の原理 (法線マッハ数のゼロクロス)

圧力勾配 ∇p を衝撃波面の法線方向とみなすと、その法線方向の流入マッハ数は

```
Mn = M · cos(α),   cos(α) = (V·∇p) / (|V|·|∇p|)
```

衝撃波を横切ると `Mn` は上流で 1 より大、下流で 1 より小となり、
衝撃波内部で必ず `Mn = 1` を横切ります (Rankine-Hugoniot の帰結)。
この **Mn=1 のゼロクロス**を検出します。判定境界が物理的に `Mn=1` で
固定されるため、Ducros のような経験的閾値が要りません。
膨張波は流れが低圧側へ向かう (`cos(α)<0`) ため自然に除外されます。

## 設定ファイル (config.yaml)

```yaml
# 入力ファイル (null = 自動検出)
input_file: null

# 複数の .vtu があった場合 (first / all)
multiple_files: first

# 比熱比 (空気なら 1.4)。音速 a = sqrt(gamma·p/rho) に使う
gamma: 1.4

# ゼロ割回避用の微小量
eps: 1.0e-12

# 出力ファイル名 (null = 自動生成 flow.vtu -> flow_ks.vtu)
output_file: null
```

Ducros と違い `threshold` 設定はありません (KS は閾値不要のため)。

## 出力

入力メッシュに以下の節点データを追加した `<名前>_ks.vtu` を生成します。

| フィールド名     | 内容                                              |
| ---------------- | ------------------------------------------------- |
| `ks_phi`         | 衝撃波らしさ (0～1)。Mn=1 ゼロクロスの強さ        |
| `shock_mask`     | 衝撃波判定 (0/1)                                  |
| `mach`           | マッハ数                                          |
| `supersonic`     | 超音速かつ評価対象の節点 (1) / 評価外 (0)         |

ParaView では、まず `supersonic` で超音速ポケットを確認し、その内部で
`ks_phi` を色付けすると衝撃波の位置が見えます。

実行時のコンソール出力例:

```
--- flow.vtu ---
  節点数            : 4900
  超音速節点数       : 2450  (M>1 のみ評価対象)
  マッハ数の範囲     : [0.350, 2.000]
  衝撃波判定節点数   : 70
  出力               : flow_ks.vtu
完了 (ParaView で ks_phi / shock_mask / mach を可視化)
```

## トラブルシューティング

### 「圧力場が見つかりません」「密度場が見つかりません」

KS センサーはマッハ数の計算に圧力と密度が必須です。SU2 の `VOLUME_OUTPUT` に
`PRESSURE` と `DENSITY` を追加して再出力してください。フィールド名が違う場合は
`load_su2_vtu` の探索候補 (`Pressure`/`Density` など) を調整します。

### 衝撃波が全く検出されない

- 流れ場に超音速領域 (M>1) が存在するか `mach` フィールドで確認
- 完全な亜音速流では KS センサーは原理的に何も検出しません (正常な挙動)
- `gamma` が流体に合っているか確認 (空気は 1.4)

### 検出が弱い / ノイズが多い

- 圧力勾配がほぼ無い一様領域は方向が不定なため、`gradp_ref` の 1% 未満を
  数値的に保護しています (経験的衝撃波閾値ではありません)。格子が極端に粗い、
  または流れ場が未収束だと勾配が乱れることがあります。

## 注意点

- **適用範囲**: 超音速領域限定。遷音速翼の亜音速側の衝撃波足元などは取りこぼす可能性があります。
- **本実装の位置づけ**: KS 法の核心 (特性線収束 = 法線マッハ数の音速横断) を再現した
  実用版です。原論文は接触不連続と膨張波の厳密な区別や、より精緻な特性線追跡を含みます。
  完全な追試には原論文 (下記) を参照してください。
- **勾配精度**: 非構造格子上の近傍最小二乗による再構成です。SU2 が直接出力する勾配を
  使う方が精度・速度の面で有利です。
