# 衝撃波センサ実行後の VTU ファイルと ParaView 操作ガイド

このドキュメントは、`ducros_sensor.py` および `lovely_sensor.py` を実行して得られる VTU ファイルについて、
**(A) ファイルにどのような変化が加えられるか** と
**(B) ParaView でどう操作すれば何が確認できるか** を網羅的にまとめたものです。

- **第1〜3部**: Ducros センサー (`*_ducros.vtu`)
- **第4部**: Lovely-Haimes センサー (`*_lovely.vtu`)

---

## 第1部: センサ実行後に VTU ファイルに加わる変化

### 1.1 全体像

センサは入力ファイル (例: `flow.vtu`) を **読み込むだけで上書きはせず** 、
新しいファイル `flow_ducros.vtu` を生成します。
このとき、**メッシュ形状 (節点座標・セル接続) は一切変更されません** 。
変わるのは「節点データ (point data) に 4 つのスカラー場が追加される」点だけです。

```
flow.vtu  (入力)                    flow_ducros.vtu  (出力)
├─ 節点座標         ────────────→   ├─ 節点座標         (変化なし)
├─ セル接続         ────────────→   ├─ セル接続         (変化なし)
└─ point_data                       └─ point_data
   └─ Velocity 等  ────────────→      ├─ Velocity 等   (変化なし)
                                       ├─ ducros_phi    ★追加
                                       ├─ divergence    ★追加
                                       ├─ vorticity_z   ★追加
                                       └─ shock_mask    ★追加
```

### 1.2 追加される 4 つのフィールド

| フィールド名   | 型      | 範囲     | 物理的意味                          | 主な用途                  |
| ------------- | ------- | -------- | ----------------------------------- | ------------------------- |
| `ducros_phi`  | float   | 0 〜 1   | Ducros センサー値                   | 衝撃波の連続的な強度マップ |
| `divergence`  | float   | 任意     | 速度の発散 ∇·u                      | 圧縮/膨張の確認 (検証用)  |
| `vorticity_z` | float   | 任意     | 渦度 ω_z                            | 渦領域の確認 (検証用)     |
| `shock_mask`  | int     | 0 or 1   | 衝撃波ラベル (phi ≥ threshold)      | 二値の衝撃波領域          |

### 1.3 各フィールドの詳しい意味

**`ducros_phi`** — このスクリプトの主役。
`max(-∇·u, 0) / (|∇·u| + |ω_z| + eps)` で計算される 0〜1 の値。
1 に近いほど「強い圧縮かつ渦が小さい」= 衝撃波らしい領域。
0 に近いほど「渦優位」または「滑らかな流れ」。

**`divergence`** (∇·u) — 速度の発散。
衝撃波では流れが急減速して圧縮されるため **大きな負の値** を取る。
膨張波では正。センサが衝撃波を拾った理由の裏付けに使う。

**`vorticity_z`** (ω_z) — 2D の渦度。
境界層・後流・渦で大きな値を取る。
ここが大きいと `ducros_phi` の分母が増えてセンサ値が抑えられる
(= 渦を衝撃波と誤検知しにくくする仕組み) 。

**`shock_mask`** — `ducros_phi >= threshold` を満たす節点を 1、それ以外を 0 とした二値ラベル。
`config.yaml` の `threshold` (既定 0.95) で境界が決まる。
セグメンテーションの参照ラベルとして直接使えるのはこれ。

## 第2部: ParaView 操作ガイド

### 2.1 ファイルを開く

1. ParaView を起動
2. **File → Open** で `flow_ducros.vtu` を選択
3. 左下の **Properties** パネルで **`Apply`** ボタン (緑) を押す

`Apply` を押すまで何も表示されないので注意。

### 2.2 基本: 色付け対象を切り替える

画面上部のツールバーに **色付け対象のプルダウン** があります
(初期値は "Solid Color")。
ここでフィールドを選ぶと、その値で色付け表示されます。

センサ実行後は、元のフィールド (Velocity 等) に加えて
**`ducros_phi` / `divergence` / `vorticity_z` / `shock_mask` の 4 つが追加** されています。

---

### 2.3 操作パターン別ガイド

#### パターン A: 衝撃波の強度マップを見る (連続値)

| 操作 | 内容 |
| ---- | ---- |
| 1 | 色付けプルダウンで **`ducros_phi`** を選択 |
| 2 | **`Edit Color Map`** (カラーバー編集アイコン) を開く |
| 3 | **`Rescale to Custom Range`** で範囲を **0〜1** に固定 |
| 4 | カラーマップを `Viridis` や `Black-Body Radiation` に変更 (任意) |

**確認できること**: 衝撃波がどこに、どのくらいの強度で立っているか。
1 付近 (暖色) が衝撃波。誤検知の有無や強度分布が一目で分かる。

#### パターン B: 衝撃波ラベルだけを抽出する (二値)

| 操作 | 内容 |
| ---- | ---- |
| 1 | パイプラインで `flow_ducros.vtu` を選択 |
| 2 | **Filters → Common → Threshold** を選択 |
| 3 | `Scalars` に **`shock_mask`** を指定 |
| 4 | 範囲を **下限 1・上限 1** に設定 |
| 5 | **`Apply`** を押す |

**確認できること**: `shock_mask == 1` の領域 = 衝撃波と判定された節点だけが残る。
これがセグメンテーションのラベルそのもの。翼のどこに衝撃波があるか明確に見える。

#### パターン C: 閾値を変えながら衝撃波領域を探る (再実行なし)

| 操作 | 内容 |
| ---- | ---- |
| 1 | **Filters → Common → Threshold** を選択 |
| 2 | `Scalars` に **`ducros_phi`** を指定 (shock_mask ではない) |
| 3 | 下限スライダーを動かして 0.8 / 0.9 / 0.95 等を試す |
| 4 | `Apply` で結果を確認 |

**確認できること**: スクリプトを再実行せずに閾値の感度を試せる。
適切な閾値が決まったら `config.yaml` の `threshold` に反映して再実行する。

#### パターン D: センサの判定根拠を検証する

| 操作 | 内容 |
| ---- | ---- |
| 1 | 色付けプルダウンで **`divergence`** を選択 |
| 2 | カラーマップで負の値 (圧縮) が分かるよう範囲調整 |
| 3 | 別ビューで **`vorticity_z`** も表示して比較 |

**確認できること**:
衝撃波の位置で `divergence` が強く負 → 圧縮が起きている裏付け。
後流・境界層で `vorticity_z` が大きい → そこを衝撃波と誤検知していないかの確認。
`ducros_phi` が高いのに `divergence` が負でない箇所があれば、センサの誤作動を疑える。

#### パターン E: 元の翼形状と重ねて位置を把握する

| 操作 | 内容 |
| ---- | ---- |
| 1 | `flow.vtu` (元ファイル) も **File → Open** で開く |
| 2 | 元ファイルを背景として表示し、`Opacity` を 0.3 程度に下げる |
| 3 | その上にパターン B の Threshold 結果 (shock_mask) を重ねる |
| 4 | 必要なら **Filters → Alphabetical → Extract Surface** で翼表面を抽出 |

**確認できること**: 衝撃波ラベルが翼面のどこ (前縁/後縁/上面/下面) に対応するか。
位置関係が明確になり、物理的な妥当性を判断しやすい。

---

### 2.4 さらに便利なフィルタ (応用)

| フィルタ | 用途 |
| -------- | ---- |
| **Contour** | `ducros_phi = 0.95` の等値線を引き、衝撃波の輪郭を線として可視化 |
| **Clip** | 領域を切断し、内部の衝撃波構造を断面で確認 |
| **Calculator** | 既存フィールドから新しい量を計算 (例: `divergence` の絶対値) |
| **Plot Over Line** | 翼面に垂直な線上で `ducros_phi` の変化をグラフ化 |
| **Spreadsheet View** | フィールド値を表形式で確認。`shock_mask` の合計で衝撃波節点数を数えられる |

## 第3部: 確認項目チェックリスト

センサの妥当性を確認する際の推奨手順です。

- [ ] **パターン A** で `ducros_phi` を表示し、衝撃波が想定位置に立っているか
- [ ] 衝撃波以外 (よどみ点・後流・境界層) で `ducros_phi` が高くなる誤検知がないか
- [ ] **パターン D** で `divergence` が衝撃波位置で負になっているか (物理的裏付け)
- [ ] **パターン B** で `shock_mask` を抽出し、ラベルが広すぎ/狭すぎないか
- [ ] 不適切なら **パターン C** で閾値を探り、`config.yaml` に反映して再実行
- [ ] **パターン E** で翼形状と重ね、衝撃波位置が物理的に妥当か

---

## 補足: トラブル時の確認

| 症状 | 確認・対処 |
| ---- | ---------- |
| プルダウンに 4 フィールドが出ない | `Apply` を押したか / 出力ファイル (`_ducros.vtu`) を開いているか |
| `ducros_phi` が全体的に低い | 流れに衝撃波がない可能性 / 速度場の読み込みが正しいか |
| `ducros_phi` が広範囲で高い | 渦度補正が効いていない可能性 / `divergence` を確認 |
| `shock_mask` が空 | `threshold` が高すぎる → パターン C で適正値を探る |
| 色が単色のまま | 色付けプルダウンが "Solid Color" のまま → フィールドを選択 |

## 第4部: Lovely-Haimes センサー (`lovely_sensor.py`)

### 4.1 出力ファイルに追加されるフィールド

センサは `*_lovely.vtu` を生成し、以下の 4 フィールドを point data に追加します。

```
flow.vtu  (入力)                    flow_lovely.vtu  (出力)
├─ 節点座標         ────────────→   ├─ 節点座標         (変化なし)
├─ セル接続         ────────────→   ├─ セル接続         (変化なし)
└─ point_data                       └─ point_data
   └─ Velocity 等  ────────────→      ├─ Velocity 等   (変化なし)
                                       ├─ normal_mach   ★追加
                                       ├─ grad_p_mag    ★追加
                                       ├─ sound_speed   ★追加
                                       └─ shock_mask    ★追加
```

| フィールド名   | 型    | 範囲     | 物理的意味                                         | 主な用途                     |
| ------------- | ----- | -------- | -------------------------------------------------- | ---------------------------- |
| `normal_mach` | float | 任意     | 法線マッハ数 Mn = (u·∇p) / (a\|∇p\|)              | **Mn=1 等値線で衝撃波を描画** |
| `grad_p_mag`  | float | ≥ 0      | 圧力勾配の大きさ \|∇p\|                            | 衝撃波の強度・位置の確認      |
| `sound_speed` | float | ≥ 0      | 局所音速 a = √(γp/ρ)                              | 流れ場の検証用                |
| `shock_mask`  | int   | 0 or 1   | Mn ≥ threshold を満たす節点を 1 に（`compression_filter: true` のとき u·∇p > 0 も追加条件） | 二値の衝撃波ラベル |

**`normal_mach` の符号の意味**:
- 正 (Mn > 0): 速度が圧力上昇方向を向いている = 圧縮
- 負 (Mn < 0): 速度が圧力降下方向を向いている = 膨張
- **Mn = 1 の等値線が衝撃波の位置** (法線方向の流速が音速と等しい)

---

### 4.2 ParaView 操作パターン

#### パターン F: Mn=1 等値線で衝撃波断面を見る (推奨)

| 手順 | 操作 |
| ---- | ---- |
| 1 | `NACA0012_M0.95_lovely.vtu` を **File → Open** で開き、`Apply` |
| 2 | パイプラインで vtu を選択した状態で **Filters → Common → Contour** を開く |
| 3 | **Contour By** のプルダウンで **`normal_mach`** を選択 |
| 4 | **Isosurfaces** の値欄に **`1.0`** を入力 |
| 5 | `Apply` を押す → Mn=1 の等値線が衝撃波位置に表示される |

**確認できること**: Ducros センサーのパターン A と異なり、等値線は「Mn が正確に 1 になる位置」を1本の線で示す。
衝撃波の向き・傾き・翼上下面での位置が視覚的に明確になる。

> **Tip**: 元の流れ場 (`flow.vtu`) を背景表示し、`Pressure` で色付けしておくと、圧力上昇と等値線の対応がひと目で確認できる。

#### パターン G: shock_mask で衝撃波節点のみ抽出する

| 手順 | 操作 |
| ---- | ---- |
| 1 | **Filters → Common → Threshold** を選択 |
| 2 | `Scalars` に **`shock_mask`** を指定 |
| 3 | 範囲を **下限 1・上限 1** に設定 |
| 4 | `Apply` → Mn ≥ 1.0 かつ u·∇p > 0 の節点のみ残る |

**確認できること**: `shock_mask=1` は膨張波を除外した上で衝撃波と判定された節点だけ。
パターン F の等値線と重ねると、連続的な等値線と離散的なマスクの対応を確認できる。

#### パターン H: 等値線と shock_mask を重ねて比較する

| 手順 | 操作 |
| ---- | ---- |
| 1 | パターン F の Contour を表示した状態のまま |
| 2 | パイプラインの vtu を再選択し、パターン G の Threshold も追加 |
| 3 | Threshold の色を赤、Contour を黒など対比色に設定 |
| 4 | 両方を同時に表示する |

**確認できること**: 等値線 (理論値 Mn=1) とマスク節点 (閾値処理後) の一致度。
等値線から外れたマスク節点は `mn_threshold` の調整候補。

#### パターン I: Mn の分布をカラーマップで確認する

| 手順 | 操作 |
| ---- | ---- |
| 1 | 色付けプルダウンで **`normal_mach`** を選択 |
| 2 | **Edit Color Map** → `Rescale to Custom Range` で範囲を **-1.5 〜 1.5** に設定 |
| 3 | カラーマップを **`Cool to Warm`** (青=負, 白=0, 赤=正) に変更 |
| 4 | カラーバーに **Add Range Labels** または手動で 1.0 のマーカーを追加 |

**確認できること**: 衝撃波 (Mn>1 の赤い領域) と膨張波 (Mn<0 の青い領域) の空間分布。
衝撃波前後のマッハ数変化の鋭さから、衝撃波の厚みが数値解でどう再現されているかも確認できる。

### 4.3 Ducros センサーとの比較

| 観点             | Ducros センサー (`ducros_phi`)            | Lovely-Haimes センサー (`normal_mach`)          |
| ---------------- | ----------------------------------------- | ----------------------------------------------- |
| 等値線の描き方   | `ducros_phi = 0.95` 等 (閾値は経験的)     | **`normal_mach = 1.0`** (物理的に明確な基準)    |
| 膨張波の扱い     | 分母の渦度で自動抑制                      | `u·∇p > 0` フィルタで除外 (`shock_mask` 側)    |
| 渦領域での誤検知 | 渦度が大きいと `ducros_phi` が下がり抑制  | 渦のある領域でも Mn が大きければ拾う可能性あり   |
| 推奨用途         | 衝撃波の強度マップ・閾値感度の確認        | **衝撃波位置の正確な等値線可視化**               |

### 4.4 トラブル時の確認 (Lovely-Haimes 固有)

| 症状 | 確認・対処 |
| ---- | ---------- |
| `normal_mach` フィールドが出ない | `Apply` を押したか / `_lovely.vtu` を開いているか |
| Contour に何も表示されない | Mn の最大値を確認 (M=1 以下の流れでは等値線が存在しない) |
| `shock_mask` が空 | `mn_threshold` が高すぎる → `config.yaml` で `1.0` → `0.95` に下げて再実行 |
| 等値線が翼全体に広がる | `normal_mach` に異常値あり → `grad_p_mag` を確認し、圧力勾配がゼロに近い領域を疑う |
| Mn=1 等値線が複数本出る | 数値拡散で衝撃波が鈍っている → 正常。最も外側の線が真の衝撃波位置に近い |
